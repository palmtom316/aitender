# aitender 项目需求文件（实施基线版）

> 版本：`v1.1`
> 日期：`2026-03-11`
> 状态：Phase 1 MVP 已形成首条可验证闭环

## 1. 文档目的

本文档用于把 `aitender` 当前已确认的产品边界和已落地的 Phase 1 MVP 能力对齐到同一份基线文档中，避免需求文件与实现状态脱节。

## 2. 产品定位

`aitender` 是一个面向电力施工企业投标团队的 AI 辅助投标作业平台。当前总范围仍包含 6 个模块：

1. 投标资料库
2. 招标分析
3. 标书编制
4. 标书审核
5. 标书排版
6. 标书管理

本阶段只有“投标资料库”进入详细实现，且只聚焦“规范规程 PDF”链路。

## 3. Phase 1 已确认范围

- 支持单组织、项目隔离和基础角色。
- 支持规范 PDF 上传、原件保存和文档版本记录。
- 支持可插拔 OCR 适配层。
- 支持 OCR 中间产物标准化和调试工件保存。
- 支持规范条款树、条文说明映射和结构校验。
- 支持按关键词检索条款并返回页码、摘要、树路径和说明摘要。
- 支持前端查看文档列表、处理状态、检索结果、详情和高亮预览。
- 支持查询处理任务状态、失败原因和关键处理步骤日志。

## 4. Phase 1 MVP 交付结果

截至 `2026-03-11`，本分支已经具备一条可验证的最小闭环：

1. 上传规范 PDF。
2. 创建文档、版本和原始工件记录。
3. 调用 OCR 处理任务并生成任务状态与审计日志。
4. 将 OCR 结果标准化为统一内部结构。
5. 生成规范条款索引和检索结果。
6. 在 Web 端展示处理状态、检索结果、条款详情和高亮预览。

## 5. 当前实现边界

以下能力已经实现：

- 登录页、项目页、资料库页和规范详情页的最小界面。
- `MinerU / 商业 OCR` 的适配入口和统一调度层。
- 规范正文条款索引、条文说明映射、结构校验和检索 API。
- 处理任务查询 API：可返回 `pending/running/completed/failed` 状态及日志。
- Web 侧资料库主流程测试和轻量级端到端测试入口。

以下能力仍属于后续阶段：

- 真实数据库持久化、PostgreSQL FTS 正式接线。
- 规范条文说明从真实来源自动抽取，而不是测试/演示数据注入。
- 其他资料类型的完整入库处理链路。
- 招标分析、标书编制、审核、排版和管理模块的详细实现。

## 6. 验证基线

当前 MVP 应至少满足以下验证：

- `PYTHONPATH=services/api-server .venv/bin/pytest services/api-server/tests -q`
- `PYTHONPATH=services/api-server .venv/bin/pytest services/api-server/tests/test_norm_pipeline_e2e.py -q`
- `npm --prefix apps/web test -- --runInBand`
- `npm --prefix apps/web run test:e2e`
- `npm --prefix apps/web run build`

## 7. 后续阶段

下一阶段重点是把当前基于内存和演示数据的链路继续替换为真实持久化与异步流水线，包括：

- 文档、条款、任务和审计日志的数据库化。
- 真实 OCR 供应商接线与回放调试能力。
- 规范资料库对“招标分析 / 标书编制 / 标书审核”模块的接口输出。
